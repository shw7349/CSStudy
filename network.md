# 🌐 네트워크 (Network) - CS 문제은행

> 총 100문제 | Level 1~4

---

## 📖 주요 CS 용어 정리

### OSI 계층 & 프로토콜
- **OSI 7계층**: 네트워크 통신을 7개 계층으로 분리한 참조 모델 (물리/데이터링크/네트워크/전송/세션/표현/응용)
- **TCP/IP**: 인터넷 표준 프로토콜. 4계층 구조 (네트워크 인터페이스/인터넷/전송/응용)
- **프로토콜(Protocol)**: 네트워크 통신을 위한 규칙과 약속
- **패킷(Packet)**: 네트워크에서 데이터를 전송하는 기본 단위
- **캡슐화(Encapsulation)**: 상위 계층 데이터에 헤더를 추가하는 과정

### IP & 주소 체계
- **IP(Internet Protocol)**: 네트워크 계층 프로토콜. 패킷을 목적지까지 전달
- **IPv4**: 32비트 주소 체계 (예: 192.168.0.1)
- **IPv6**: 128비트 주소 체계 (IPv4 주소 고갈 문제 해결)
- **서브넷(Subnet)**: IP 주소를 논리적으로 분할한 네트워크
- **서브넷 마스크(Subnet Mask)**: 네트워크 부분과 호스트 부분을 구분하는 값
- **NAT(Network Address Translation)**: 사설 IP를 공인 IP로 변환하는 기술

### 전송 계층
- **TCP(Transmission Control Protocol)**: 연결 지향, 신뢰성 보장 프로토콜
- **UDP(User Datagram Protocol)**: 비연결형, 빠르지만 신뢰성 없는 프로토콜
- **포트(Port)**: 애플리케이션을 구분하는 논리적 통신 끝점 (0-65535)
- **소켓(Socket)**: IP주소 + 포트 번호로 구성된 통신 종단점
- **3-Way Handshake**: TCP 연결 수립 과정 (SYN → SYN-ACK → ACK)
- **4-Way Handshake**: TCP 연결 종료 과정

### 응용 계층 프로토콜
- **HTTP(HyperText Transfer Protocol)**: 웹 통신 프로토콜 (80번 포트)
- **HTTPS**: HTTP + SSL/TLS 암호화 (443번 포트)
- **DNS(Domain Name System)**: 도메인 이름을 IP 주소로 변환하는 시스템
- **FTP(File Transfer Protocol)**: 파일 전송 프로토콜
- **SMTP(Simple Mail Transfer Protocol)**: 이메일 전송 프로토콜

### 라우팅 & 스위칭
- **라우터(Router)**: 서로 다른 네트워크 간 패킷을 전달하는 장비 (3계층)
- **스위치(Switch)**: 같은 네트워크 내에서 프레임을 전달하는 장비 (2계층)
- **라우팅(Routing)**: 패킷의 최적 경로를 선택하는 과정
- **라우팅 테이블**: 목적지까지의 경로 정보를 저장한 테이블
- **MAC 주소**: 네트워크 인터페이스의 물리적 주소 (48비트)

### 네트워크 보안 & 기타
- **방화벽(Firewall)**: 네트워크 트래픽을 제어하여 보안을 강화하는 장비/소프트웨어
- **프록시(Proxy)**: 클라이언트와 서버 사이의 중계 서버
- **로드 밸런서(Load Balancer)**: 트래픽을 여러 서버에 분산시키는 장비
- **CDN(Content Delivery Network)**: 콘텐츠를 지리적으로 분산된 서버에 캐싱
- **VPN(Virtual Private Network)**: 공용 네트워크를 통해 안전한 사설 네트워크 구축
- **대역폭(Bandwidth)**: 단위 시간당 전송할 수 있는 데이터량
- **지연시간(Latency)**: 데이터 전송에 걸리는 시간

### 🎯 시니어급 고급 용어

#### TCP 고급 메커니즘
- **TCP Fast Open(TFO)**: 3-Way Handshake 중에 데이터를 함께 전송하여 지연 시간 단축
- **TCP Keepalive**: 유휴 연결이 살아있는지 주기적으로 확인하는 메커니즘
- **Nagle 알고리즘**: 작은 패킷들을 모아서 전송하여 네트워크 효율 향상
- **Delayed ACK**: ACK를 즉시 보내지 않고 지연시켜 ACK 패킷 수를 줄이는 기법
- **Selective ACK(SACK)**: 누락된 특정 세그먼트만 재전송하도록 하는 TCP 확장
- **Window Scaling**: TCP 윈도우 크기를 64KB 이상으로 확장하는 옵션 (고속 네트워크)
- **TIME_WAIT**: TCP 연결 종료 후 일정 시간 대기하여 지연된 패킷 처리

#### 혼잡 제어(Congestion Control)
- **AIMD(Additive Increase Multiplicative Decrease)**: 혼잡 윈도우를 선형 증가, 지수 감소시키는 방식
- **Slow Start**: 연결 초기에 혼잡 윈도우를 지수적으로 증가
- **Congestion Avoidance**: 혼잡 임계값 도달 후 선형적으로 증가
- **Fast Retransmit**: 3개의 중복 ACK 수신 시 타임아웃 전에 재전송
- **Fast Recovery**: 패킷 손실 후 Slow Start 대신 혼잡 회피 단계로 진입
- **CUBIC**: Linux 기본 혼잡 제어 알고리즘. 대역폭-지연 곱이 큰 네트워크에 최적화
- **BBR(Bottleneck Bandwidth and RTT)**: Google이 개발한 최신 혼잡 제어 알고리즘

#### HTTP 고급
- **HTTP/2**: 멀티플렉싱, 서버 푸시, 헤더 압축(HPACK) 지원
- **HTTP/3**: QUIC 프로토콜 기반. UDP 위에서 동작하며 연결 수립이 빠름
- **QUIC**: UDP 기반의 전송 프로토콜. 0-RTT 연결, 멀티플렉싱, 암호화 내장
- **Server-Sent Events(SSE)**: 서버가 클라이언트에게 단방향 실시간 이벤트 전송
- **WebSocket**: 양방향 실시간 통신을 위한 프로토콜 (HTTP 업그레이드)

#### 라우팅 프로토콜
- **BGP(Border Gateway Protocol)**: 인터넷의 AS(자율 시스템) 간 라우팅 프로토콜
- **OSPF(Open Shortest Path First)**: 링크 상태 기반 동적 라우팅 프로토콜
- **RIP(Routing Information Protocol)**: 거리 벡터 기반 라우팅 (최대 15홉)
- **Anycast**: 가장 가까운 서버로 라우팅되는 주소 지정 방식 (DNS, CDN에 사용)
- **Multicast**: 하나의 송신자가 특정 그룹에게만 데이터 전송

#### 네트워크 가상화
- **VLAN(Virtual LAN)**: 물리적 네트워크를 논리적으로 분할 (2계층)
- **VXLAN(Virtual Extensible LAN)**: VLAN을 확장한 오버레이 네트워크 (최대 1600만 개)
- **Overlay Network**: 기존 네트워크 위에 구축된 가상 네트워크
- **Underlay Network**: Overlay를 지원하는 물리적 네트워크 인프라
- **SDN(Software-Defined Networking)**: 네트워크 제어를 소프트웨어로 중앙 집중화
- **NFV(Network Functions Virtualization)**: 네트워크 기능을 가상화하여 범용 하드웨어에서 실행

#### 패킷 분석 & 성능
- **MTU(Maximum Transmission Unit)**: 한 번에 전송할 수 있는 최대 패킷 크기 (일반적으로 1500B)
- **MSS(Maximum Segment Size)**: TCP 세그먼트의 최대 데이터 크기 (MTU - IP헤더 - TCP헤더)
- **Path MTU Discovery**: 경로상 최소 MTU를 찾아 단편화를 방지
- **Fragmentation**: IP 패킷이 MTU를 초과할 때 작은 조각으로 분할
- **Jumbo Frame**: 표준 이더넷보다 큰 프레임 (최대 9000B). 데이터센터에서 성능 향상
- **TTL(Time To Live)**: 패킷이 라우터를 통과할 때마다 감소. 0이 되면 폐기 (무한 루프 방지)
- **RTT(Round-Trip Time)**: 패킷이 목적지까지 갔다가 돌아오는 시간

#### 보안 & 공격
- **SYN Flood**: SYN 패킷만 대량 전송하여 서버 자원 고갈시키는 DDoS 공격
- **ARP Spoofing**: MAC 주소를 속여 트래픽을 가로채는 공격
- **DNS Amplification**: DNS 응답을 증폭시켜 대상에게 트래픽을 집중시키는 DDoS 공격
- **Man-in-the-Middle(MITM)**: 통신 중간에서 데이터를 가로채거나 조작
- **TLS/SSL Handshake**: 암호화 통신을 위한 인증서 교환 및 키 협상 과정
- **Zero Trust Network**: 모든 네트워크 트래픽을 신뢰하지 않고 검증하는 보안 모델

#### 대규모 시스템 네트워킹
- **Connection Pooling**: 연결을 재사용하여 핸드셰이크 오버헤드 제거
- **Keep-Alive**: HTTP 연결을 유지하여 여러 요청을 재사용
- **L4/L7 로드 밸런싱**: 전송 계층(포트) vs 응용 계층(URL) 기반 부하 분산
- **Direct Server Return(DSR)**: 응답을 로드 밸런서를 거치지 않고 클라이언트에 직접 전송
- **Health Check**: 백엔드 서버의 상태를 주기적으로 확인하여 장애 서버 제외
- **Graceful Degradation**: 네트워크 장애 시 서비스 수준을 단계적으로 낮춤
- **Circuit Breaker**: 장애 서비스 호출을 차단하여 연쇄 장애 방지
- **Bulkhead Pattern**: 리소스를 격리하여 한 영역의 장애가 전체로 확산되지 않도록 방지
- **Retry with Backoff**: 실패 시 지수 백오프로 재시도하여 부하 분산
- **Rate Limiting**: 요청 속도를 제한하여 서버 과부하 방지 (Token Bucket, Leaky Bucket)
- **Idempotency**: 동일한 요청을 여러 번 수행해도 결과가 같도록 보장 (네트워크 재시도 안전성)
- **Request Coalescing**: 동일한 요청을 하나로 합쳐서 백엔드 부하 감소

### Q1. OSI 7계층이란?

#### A. 핵심 요약
네트워크 통신을 7개 계층으로 분리한 참조 모델입니다.

#### 상세 설명
- 물리 → 데이터링크 → 네트워크 → 전송 → 세션 → 표현 → 응용
- 각 계층은 독립적으로 동작
- 캡슐화/역캡슐화

#### 실무 예시
- 문제 해결 시 계층별 분석
- 장비: 허브(1), 스위치(2), 라우터(3)

#### 면접 답변 예시
**1분**: "OSI는 네트워크를 7계층으로 나눠 각 계층이 독립적으로 동작합니다."

**3분**: "실제로는 TCP/IP 4계층을 많이 사용합니다. 트러블슈팅 시 계층별로 문제를 분리합니다."

#### 꼬리 질문
- TCP/IP 4계층과 매핑?
- 각 계층 대표 프로토콜?

### Q2. TCP와 UDP 차이는?

#### A. 핵심 요약
TCP는 신뢰성 보장, UDP는 빠른 전송을 위한 프로토콜입니다.

#### 상세 설명
- TCP: 연결 지향, 순서 보장, 흐름/혼잡 제어
- UDP: 비연결, 빠름, 손실 가능
- 헤더 크기: TCP 20B, UDP 8B

#### 실무 예시
- TCP: 웹, 이메일, 파일 전송
- UDP: DNS, 스트리밍, 게임

#### 면접 답변 예시
**1분**: "TCP는 신뢰성, UDP는 속도가 중요할 때 사용합니다."

**3분**: "TCP의 3-way handshake, 재전송, 혼잡 제어로 오버헤드가 있습니다. UDP는 애플리케이션이 신뢰성을 처리합니다."

#### 꼬리 질문
- 3-way handshake?
- QUIC 프로토콜?

### Q3. IP 주소 체계란?

#### A. 핵심 요약
네트워크 상 장치를 식별하는 논리적 주소입니다.

#### 상세 설명
- IPv4: 32비트, 약 43억 개
- IPv6: 128비트
- 서브넷, CIDR

#### 실무 예시
- 사설 IP vs 공인 IP
- NAT

#### 면접 답변 예시
**1분**: "IPv4는 32비트 주소로 고갈 문제가 있어 IPv6와 NAT를 사용합니다."

**3분**: "CIDR로 유연한 서브넷팅, 사설 IP로 주소 절약, NAT로 공인 IP 공유합니다."

#### 꼬리 질문
- 사설 IP 대역?
- CIDR 표기법?

### Q4. HTTP란?

#### A. 핵심 요약
웹에서 클라이언트-서버 간 데이터를 주고받는 프로토콜입니다.

#### 상세 설명
- 요청-응답 구조
- 무상태(Stateless)
- 메서드: GET, POST, PUT, DELETE

#### 실무 예시
- REST API
- 웹 브라우저

#### 면접 답변 예시
**1분**: "HTTP는 요청-응답 기반 무상태 프로토콜로, 메서드로 동작을 구분합니다."

**3분**: "무상태 특성상 세션/쿠키로 상태를 관리합니다. HTTP/2, HTTP/3으로 성능이 개선되었습니다."

#### 꼬리 질문
- HTTP 메서드 의미?
- HTTP/1.1 vs HTTP/2?

### Q5. HTTPS란?

#### A. 핵심 요약
HTTP에 TLS 암호화를 추가한 보안 프로토콜입니다.

#### 상세 설명
- TLS 핸드셰이크
- 인증서로 서버 신원 확인
- 대칭키로 데이터 암호화

#### 실무 예시
- 모든 웹 서비스 기본
- Let's Encrypt

#### 면접 답변 예시
**1분**: "HTTPS는 TLS로 암호화하여 도청과 변조를 방지합니다."

**3분**: "비대칭키로 대칭키를 교환하고, 대칭키로 데이터를 암호화합니다. 인증서 체인으로 신뢰를 확인합니다."

#### 꼬리 질문
- TLS 핸드셰이크 과정?
- 인증서 체인?

### Q6. DNS란?

#### A. 핵심 요약
도메인 이름을 IP 주소로 변환하는 시스템입니다.

#### 상세 설명
- 계층적 구조: 루트 → TLD → 권한
- 재귀 vs 반복 쿼리
- A, AAAA, CNAME, MX 레코드

#### 실무 예시
- 도메인 설정
- DNS 캐싱

#### 면접 답변 예시
**1분**: "DNS는 도메인을 IP로 변환하며, 계층적 캐싱으로 효율적입니다."

**3분**: "로컬 → ISP → 루트 → TLD → 권한 서버 순으로 조회합니다. TTL로 캐시 시간을 관리합니다."

#### 꼬리 질문
- DNS 레코드 종류?
- DNS over HTTPS?

### Q7. TCP 3-way Handshake란?

#### A. 핵심 요약
TCP 연결 설정을 위한 SYN → SYN-ACK → ACK 과정입니다.

#### 상세 설명
- 클라이언트: SYN (seq=x)
- 서버: SYN-ACK (seq=y, ack=x+1)
- 클라이언트: ACK (ack=y+1)

#### 실무 예시
- 연결 지연 분석
- SYN Flood 공격

#### 면접 답변 예시
**1분**: "3-way handshake는 양쪽이 시퀀스 번호를 교환하여 연결을 확립합니다."
