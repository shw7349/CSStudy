### Q1. 시간 복잡도란?

#### A. 핵심 요약
입력 크기에 따른 알고리즘 실행 시간을 Big-O로 표현합니다.

#### 상세 설명
- Big-O: 최악의 경우 상한
- 상수와 낮은 차수 무시
- O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2^n)

#### 실무 예시
- 배열 조회 O(1)
- 이진 탐색 O(log n)
- 정렬 O(n log n)

#### 면접 답변 예시
**1분**: "시간 복잡도는 입력 크기 n에 대한 연산 횟수의 증가율을 Big-O로 표현합니다."

**3분**: "O(n²)과 O(n log n)의 차이는 n이 커질수록 극적입니다. n=1만일 때 O(n²)은 1억 연산, O(n log n)은 약 13만 연산입니다."

#### 꼬리 질문
- Big-O, Big-Ω, Big-Θ 차이?
- Amortized 분석이란?

### Q2. 버블 정렬(Bubble Sort)이란?

#### A. 핵심 요약
인접 요소를 비교하며 교환하는 O(n²) 정렬 알고리즘입니다.

#### 상세 설명
- 한 패스마다 최대값이 끝으로 이동
- 최선 O(n), 평균/최악 O(n²)
- 안정 정렬, 제자리 정렬

#### 실무 예시
- 교육용으로만 사용
- 거의 정렬된 소규모 데이터

#### 면접 답변 예시
**1분**: "버블 정렬은 인접 요소 비교 교환을 반복하며, O(n²)으로 비효율적입니다."

**3분**: "조기 종료 최적화로 정렬된 경우 O(n)이 됩니다. 하지만 실무에서는 퀵소트나 병합 정렬을 사용합니다."

#### 꼬리 질문
- 안정 정렬이란?
- 조기 종료 최적화란?

### Q3. 선택 정렬(Selection Sort)이란?

#### A. 핵심 요약
최소값을 찾아 앞으로 이동시키는 O(n²) 정렬입니다.

#### 상세 설명
- 매 패스마다 최소값 선택
- 항상 O(n²) 비교
- 교환 횟수 최소 O(n)

#### 실무 예시
- 교환 비용이 비교 비용보다 클 때
- 교육용

#### 면접 답변 예시

**3분**: "버블 정렬보다 교환이 적어 쓰기 비용이 큰 경우 유리합니다. 하지만 여전히 비효율적입니다."

### Q4. 삽입 정렬(Insertion Sort)이란?

#### A. 핵심 요약
요소를 정렬된 부분에 삽입하는 O(n²) 정렬입니다.

#### 상세 설명
- 정렬된 부분을 확장
- 최선 O(n), 평균/최악 O(n²)

#### 실무 예시

**3분**: "적응적 정렬로 부분 정렬된 데이터에 강합니다. Tim Sort는 작은 런에 삽입 정렬을 사용합니다."

### Q5. 퀵 정렬(Quick Sort)이란?

#### A. 핵심 요약
피벗을 기준으로 분할하여 정렬하는 O(n log n) 알고리즘입니다.

#### 상세 설명
- 분할 정복: 피벗 선택 → 분할 → 재귀
- 평균 O(n log n), 최악 O(n²)
- 제자리 정렬, 캐시 효율적

#### 실무 예시
- 대부분의 언어 표준 라이브러리
- 일반 목적 정렬

#### 면접 답변 예시
**1분**: "퀵 정렬은 피벗으로 분할하여 평균 O(n log n)이며, 캐시 효율이 좋습니다."

**3분**: "랜덤 피벗이나 중앙값의 중앙값으로 최악을 피합니다. 제자리 정렬이지만 호출 스택 O(log n) 공간이 필요합니다."

#### 꼬리 질문
- 최악의 경우를 피하는 방법?
- 3-way 퀵 정렬이란?

### Q6. 병합 정렬(Merge Sort)이란?

#### A. 핵심 요약
분할 후 병합하는 안정적인 O(n log n) 정렬입니다.

#### 상세 설명
- 항상 O(n log n) 보장
- O(n) 추가 공간 필요
- 안정 정렬

#### 실무 예시
- 연결 리스트 정렬
- 외부 정렬
- 안정성이 필요한 경우

#### 면접 답변 예시
**1분**: "병합 정렬은 분할 후 정렬된 부분을 병합하며, 항상 O(n log n)입니다."

**3분**: "추가 메모리가 필요하지만 최악 케이스가 없고 안정적입니다. 연결 리스트에서는 O(1) 공간으로 가능합니다."

#### 꼬리 질문
- 제자리 병합 정렬이 가능한가요?

### Q7. 힙 정렬(Heap Sort)이란?

#### A. 핵심 요약
힙 자료구조를 이용한 O(n log n) 제자리 정렬입니다.

#### 상세 설명
- 배열을 힙으로 변환 O(n)
- 최대값 추출 반복 O(n log n)
- 불안정, 제자리

#### 실무 예시
- 최악 O(n log n) 보장 필요 시
- Introspective Sort의 폴백

#### 면접 답변 예시
**1분**: "힙 정렬은 힙을 구성하고 루트를 반복 추출하여 O(n log n)에 정렬합니다."

**3분**: "최악에도 O(n log n)이지만 캐시 효율이 낮아 실제로는 퀵 정렬보다 느립니다. Heapify는 O(n)입니다."

#### 꼬리 질문

### Q9. 재귀와 반복의 차이는?

#### 상세 설명
- 재귀: 스택 메모리 사용
- 반복: 스택 불필요

#### 실무 예시

- 피보나치: 반복이 효율적

#### 면접 답변 예시
**1분**: "재귀는 코드가 간결하지만 스택 오버플로우 위험, 반복은 효율적입니다."

**3분**: "모든 재귀는 명시적 스택으로 반복 변환 가능합니다. 꼬리 재귀는 컴파일러가 반복으로 최적화할 수 있습니다."

#### 꼬리 질문

### Q10. 분할 정복(Divide and Conquer)이란?

#### A. 핵심 요약
문제를 작은 부분으로 분할하고, 해결 후 합치는 기법입니다.

#### 상세 설명
- Divide → Conquer → Combine
- 보통 재귀적 구현
- 마스터 정리로 복잡도 분석

#### 실무 예시
- 병합 정렬, 퀵 정렬
- 이진 탐색
- 카라츠바 곱셈

#### 면접 답변 예시
**1분**: "분할 정복은 문제를 분할, 재귀 해결, 결합하는 알고리즘 설계 기법입니다."

**3분**: "T(n) = aT(n/b) + f(n) 형태로 표현하고 마스터 정리로 복잡도를 구합니다. 병합 정렬은 T(n) = 2T(n/2) + O(n) = O(n log n)입니다."

#### 꼬리 질문
- 마스터 정리란?
- 분할 정복 vs 동적 프로그래밍?

**3분**: "기울기가 단조 감소하고 쿼리 x가 단조 증가하면 O(n)입니다. 그렇지 않으면 Li Chao Tree를 사용합니다."
