### Q1. 배열(Array)과 연결 리스트(Linked List)의 차이점은?

#### A. 핵심 요약
배열은 연속된 메모리에 데이터를 저장하고, 연결 리스트는 노드가 포인터로 연결된 구조입니다.

#### 상세 설명
- **배열**: 인덱스로 O(1) 접근, 삽입/삭제 O(n)
- **연결 리스트**: 순차 접근 O(n), 삽입/삭제 O(1)
- 배열은 캐시 지역성이 좋고, 연결 리스트는 동적 크기 조절이 용이

#### 실무 예시
- 배열: 고정 크기 버퍼, 조회 위주 데이터
- 연결 리스트: LRU 캐시, 실시간 삽입/삭제가 빈번한 경우

#### 면접 답변 예시
**1분**: "배열은 연속 메모리로 인덱스 접근이 O(1)이고, 연결 리스트는 포인터 기반으로 삽입/삭제가 O(1)입니다."

**3분**: "배열은 메모리가 연속적이라 캐시 효율이 좋고 랜덤 접근이 빠르지만, 크기 변경이 어렵습니다. 연결 리스트는 동적 크기 조절이 가능하고 삽입/삭제가 효율적이지만, 추가 메모리(포인터)가 필요하고 캐시 미스가 발생할 수 있습니다."

#### 꼬리 질문
- ArrayList와 LinkedList 중 언제 무엇을 쓰나요?
- 캐시 지역성이란 무엇인가요?

### Q2. 스택(Stack)과 큐(Queue)의 차이점은?

#### A. 핵심 요약
스택은 LIFO(후입선출), 큐는 FIFO(선입선출) 구조입니다.

#### 상세 설명
- **스택**: push/pop이 한쪽 끝에서만 발생
- **큐**: enqueue는 뒤에서, dequeue는 앞에서 발생
- 둘 다 삽입/삭제 O(1)

#### 실무 예시
- 스택: 함수 호출 스택, Undo 기능, 괄호 검사
- 큐: 작업 큐, BFS, 메시지 큐

#### 면접 답변 예시
**1분**: "스택은 LIFO로 마지막에 넣은 것이 먼저 나오고, 큐는 FIFO로 먼저 넣은 것이 먼저 나옵니다."

**3분**: "스택은 재귀, 백트래킹, DFS에 사용되며 함수 호출 관리에 핵심입니다. 큐는 작업 스케줄링, BFS, 버퍼링에 사용됩니다. 둘 다 배열이나 연결 리스트로 구현 가능합니다."

#### 꼬리 질문
- Deque는 무엇인가요?
- Priority Queue와 일반 Queue의 차이는?

### Q3. 해시 테이블(Hash Table)이란?

#### A. 핵심 요약
키를 해시 함수로 변환하여 값을 저장하는 자료구조로, 평균 O(1) 조회가 가능합니다.

#### 상세 설명
- 해시 함수가 키를 인덱스로 변환
- 충돌 발생 시 체이닝 또는 오픈 어드레싱으로 해결
- 평균 O(1), 최악 O(n)

#### 실무 예시
- HashMap, Dictionary 구현
- 캐시 시스템
- 데이터베이스 인덱싱

#### 면접 답변 예시
**1분**: "해시 테이블은 키를 해시 함수로 변환해 배열 인덱스에 매핑하여 O(1) 접근을 제공합니다."

**3분**: "해시 테이블은 키-값 저장에 최적화되어 있습니다. 좋은 해시 함수는 충돌을 최소화하고 균등 분포를 만듭니다. 충돌 해결로는 체이닝(연결 리스트)과 오픈 어드레싱(선형/이차 탐사)이 있습니다."

#### 꼬리 질문

### Q5. 이진 탐색 트리(BST)란?

#### A. 핵심 요약
왼쪽 자식 < 부모 < 오른쪽 자식 규칙을 따르는 이진 트리로, 탐색이 O(log n)입니다.

#### 상세 설명
- 중위 순회 시 정렬된 결과
- 평균 O(log n), 최악(편향) O(n)
- 삽입, 삭제, 검색 모두 동일한 시간 복잡도

#### 실무 예시
- 데이터베이스 인덱스
- 파일 시스템
- 자동 완성 기능

#### 면접 답변 예시
**1분**: "BST는 왼쪽 자식이 부모보다 작고 오른쪽이 큰 규칙으로, 이진 탐색처럼 O(log n) 검색이 가능합니다."

**3분**: "BST는 정렬된 데이터 유지와 빠른 검색을 제공합니다. 하지만 삽입 순서에 따라 편향될 수 있어, 이를 해결하기 위해 AVL, Red-Black 트리 같은 균형 트리가 사용됩니다."

#### 꼬리 질문
- BST가 편향되면 어떻게 되나요?
- AVL 트리와 Red-Black 트리의 차이는?

### Q6. 힙(Heap)이란?

#### A. 핵심 요약
완전 이진 트리 기반으로, 부모가 자식보다 항상 크거나(최대힙) 작은(최소힙) 자료구조입니다.

#### 상세 설명
- 삽입/삭제 O(log n), 최대/최소값 조회 O(1)
- 배열로 효율적 구현 가능 (부모: i/2, 자식: 2i, 2i+1)
- 우선순위 큐 구현에 사용

#### 실무 예시
- 작업 스케줄러
- 다익스트라 알고리즘
- 실시간 중앙값 계산

#### 면접 답변 예시
**1분**: "힙은 완전 이진 트리로 최대/최소값을 O(1)에 얻고, 삽입/삭제는 O(log n)입니다."

**3분**: "힙은 배열로 구현하면 메모리 효율적입니다. heapify 연산으로 배열을 O(n)에 힙으로 변환할 수 있습니다. 힙 정렬은 O(n log n)이며 추가 메모리가 필요 없습니다."

#### 꼬리 질문
- 힙과 BST의 차이는?
- 힙 정렬의 시간 복잡도는?

### Q7. 그래프(Graph)의 기본 개념은?

#### A. 핵심 요약
정점(Vertex)과 간선(Edge)으로 구성된 자료구조로, 관계를 표현합니다.

#### 상세 설명
- **방향 그래프**: 간선에 방향이 있음
- **무방향 그래프**: 양방향 이동 가능
- **가중치 그래프**: 간선에 비용이 있음
- 인접 행렬 또는 인접 리스트로 구현

#### 실무 예시
- 소셜 네트워크 (친구 관계)
- 지도/네비게이션
- 의존성 관리

#### 면접 답변 예시

**3분**: "인접 행렬은 O(1) 간선 확인이 가능하지만 O(V²) 공간이 필요합니다. 인접 리스트는 O(V+E) 공간으로 희소 그래프에 적합합니다. 그래프 탐색에는 BFS, DFS가 사용됩니다."

### Q8. DFS와 BFS의 차이는?

#### A. 핵심 요약
DFS는 깊이 우선(스택), BFS는 너비 우선(큐) 탐색입니다.

#### 상세 설명
- **DFS**: 한 경로를 끝까지 탐색 후 백트래킹
- **BFS**: 현재 정점의 모든 인접 정점 먼저 탐색
- 둘 다 O(V+E) 시간 복잡도

#### 실무 예시
- DFS: 미로 탐색, 위상 정렬, 사이클 탐지
- BFS: 최단 경로, 레벨별 탐색, 소셜 네트워크 친구 추천

#### 면접 답변 예시
**1분**: "DFS는 스택으로 깊이 우선 탐색, BFS는 큐로 너비 우선 탐색하며 최단 경로 보장이 다릅니다."

**3분**: "DFS는 재귀나 스택으로 구현하며 메모리 효율적입니다. BFS는 큐로 구현하며 무가중치 그래프에서 최단 경로를 보장합니다. 목적에 따라 선택합니다."

#### 꼬리 질문
- 언제 DFS, 언제 BFS를 사용하나요?
- DFS로 최단 경로를 구할 수 있나요?

### Q9. 시간 복잡도와 공간 복잡도란?

#### A. 핵심 요약
시간 복잡도는 실행 시간, 공간 복잡도는 메모리 사용량을 입력 크기 함수로 표현합니다.

#### 상세 설명
- Big-O: 최악의 경우
- Big-Ω: 최선의 경우
- Big-Θ: 평균적인 경우
- 상수는 무시, 최고차항만 고려

#### 실무 예시
- O(1): 해시 조회
- O(log n): 이진 탐색
- O(n): 선형 탐색
- O(n log n): 퀵/머지 정렬
- O(n²): 버블 정렬

#### 면접 답변 예시
**1분**: "시간 복잡도는 입력 크기에 따른 연산 횟수, 공간 복잡도는 필요한 메모리를 Big-O로 표현합니다."

**3분**: "복잡도 분석으로 알고리즘 효율성을 비교합니다. 실무에서는 시간-공간 트레이드오프를 고려합니다. 예를 들어 메모이제이션은 공간을 사용해 시간을 줄입니다."

#### 꼬리 질문
- O(n log n)과 O(n²)의 실제 차이는?
- 공간 복잡도가 중요한 경우는?

### Q10. 재귀(Recursion)란?

### Q11. 동적 배열(Dynamic Array)이란?

#### A. 핵심 요약
크기가 자동으로 조절되는 배열로, 용량 초과 시 더 큰 배열로 복사합니다.

#### 상세 설명
- 보통 2배씩 크기 증가
- 평균 삽입 O(1), 최악(리사이징) O(n)
- Amortized Analysis로 평균 O(1) 증명

#### 실무 예시
- Java ArrayList, Python list, C++ vector
- 크기를 미리 알 수 없는 데이터 저장

#### 면접 답변 예시
**1분**: "동적 배열은 용량 초과 시 자동 확장되며, 분할 상환 분석으로 평균 O(1) 삽입입니다."

**3분**: "동적 배열은 정적 배열의 고정 크기 문제를 해결합니다. 2배 확장 전략으로 복사 비용을 분산시켜 amortized O(1)을 달성합니다. 초기 용량 설정으로 리사이징을 줄일 수 있습니다."

#### 꼬리 질문
- Amortized Analysis란?
- 왜 2배씩 늘리나요?

### Q13. Set과 Map의 차이는?

#### A. 핵심 요약

### Q14. 문자열(String) 자료구조의 특징은?

### Q18. 인접 행렬 vs 인접 리스트?

#### A. 핵심 요약

#### 상세 설명
- 인접 행렬: 2D 배열, 간선 확인 O(1)
- 인접 리스트: 각 정점별 연결 리스트
- 밀집 그래프 vs 희소 그래프

#### 실무 예시
- 밀집: 인접 행렬 (작은 완전 그래프)
- 희소: 인접 리스트 (소셜 네트워크)

#### 면접 답변 예시
**1분**: "인접 행렬은 O(1) 간선 확인이지만 O(V²) 공간, 인접 리스트는 O(V+E) 공간으로 희소 그래프에 적합합니다."

**3분**: "밀집 그래프(E ≈ V²)는 인접 행렬이 유리하고, 희소 그래프(E << V²)는 인접 리스트가 효율적입니다. 모든 간선 순회는 인접 리스트가 O(V+E)로 더 빠릅니다."

#### 상세 설명
- String Interning: 중복 제거
- SSO: 짧은 문자열 인라인 저장
- Rope: 대용량 문자열 수정

#### 실무 예시
- 컴파일러 심볼 테이블
- 텍스트 에디터
- JSON 파서

#### 면접 답변 예시
**1분**: "SSO로 작은 문자열의 힙 할당을 피하고, 인터닝으로 중복을 제거합니다."

**3분**: "C++ std::string은 보통 15바이트까지 SSO합니다. 대용량 텍스트는 Rope나 Piece Table로 O(log n) 수정을 지원합니다."

### Q80. 자료구조 트레이드오프 분석 방법론은?

#### A. 핵심 요약
워크로드 분석, 벤치마킹, 이론적 분석을 조합하여 최적의 자료구조를 선택합니다.

#### 상세 설명
- 연산 빈도 프로파일링
- 메모리 vs 시간 트레이드오프
- 캐시 효율, 동시성 고려

#### 실무 예시
- 시스템 병목 분석
- 자료구조 마이그레이션
- 성능 튜닝

#### 면접 답변 예시
**1분**: "워크로드를 분석하고, 벤치마크로 검증하며, 이론과 실측을 모두 고려합니다."

**3분**: "이론적 복잡도만으로는 부족합니다. 캐시 미스, 분기 예측, 메모리 단편화 등 실제 하드웨어 특성을 고려합니다. A/B 테스트와 점진적 롤아웃으로 검증합니다."
