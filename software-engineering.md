### Q1. 소프트웨어 공학이란?

#### A. 핵심 요약
소프트웨어 개발에 체계적 접근방식을 적용하는 학문입니다.

#### 상세 설명
- 요구사항, 설계, 구현, 테스트, 유지보수
- 프로세스, 방법론, 도구
- 품질, 비용, 일정 관리

#### 실무 예시
- 프로젝트 관리
- 개발 프로세스

#### 면접 답변 예시
**1분**: "소프트웨어 공학은 고품질 소프트웨어를 효율적으로 개발하는 체계적 접근법입니다."

**3분**: "요구사항부터 유지보수까지 전체 수명주기를 관리합니다. 팀 협업과 프로세스가 핵심입니다."

#### 꼬리 질문
- 소프트웨어 위기란?
- 공학적 접근의 장점?

---

### Q2. SDLC(소프트웨어 개발 생명주기)란?

#### A. 핵심 요약
소프트웨어 개발의 단계별 프로세스입니다.

#### 상세 설명
- 계획 → 분석 → 설계 → 구현 → 테스트 → 유지보수
- 폭포수, 애자일, 반복 모델
- 각 단계의 산출물

#### 실무 예시
- 프로젝트 계획
- 방법론 선택

#### 면접 답변 예시
**1분**: "SDLC는 소프트웨어 개발의 단계를 정의하여 체계적으로 관리합니다."

**3분**: "프로젝트 특성에 따라 폭포수, 애자일 등 적절한 모델을 선택합니다."

#### 꼬리 질문
- 폭포수 vs 애자일?
- 각 단계 산출물?

### Q3. 애자일 방법론이란?

#### A. 핵심 요약
반복적, 점진적 개발로 변화에 빠르게 대응합니다.

#### 상세 설명
- 애자일 선언문 4가지 가치
- 스크럼, 칸반, XP
- 스프린트, 백로그

#### 실무 예시
- 스프린트 계획
- 데일리 스탠드업

#### 면접 답변 예시
**1분**: "애자일은 짧은 반복으로 피드백을 받아 변화에 적응합니다."

**3분**: "문서보다 동작하는 소프트웨어, 계획보다 변화 대응을 중시합니다."

#### 꼬리 질문
- 스크럼 역할?
- 애자일의 한계?

### Q4. 스크럼이란?

#### A. 핵심 요약
애자일 프레임워크로, 스프린트 단위로 개발합니다.

#### 상세 설명
- PO, SM, 개발팀
- 스프린트(2-4주)
- 백로그, 스탠드업, 회고

#### 실무 예시
- 스프린트 플래닝
- 데일리 스크럼

#### 면접 답변 예시
**1분**: "스크럼은 역할, 이벤트, 아티팩트로 구성된 애자일 프레임워크입니다."

**3분**: "PO가 우선순위를 정하고, 팀이 스프린트마다 인크리먼트를 제공합니다."

#### 꼬리 질문
- 스프린트 리뷰 vs 회고?
- 벨로시티란?

### Q9. CI/CD란?

#### A. 핵심 요약
지속적 통합과 배포로 빠르고 안정적인 릴리스를 합니다.

#### 상세 설명
- CI: 코드 통합, 빌드, 테스트 자동화
- CD: 배포 자동화
- 파이프라인

#### 실무 예시
- Jenkins, GitHub Actions
- 자동 배포

#### 면접 답변 예시
**1분**: "CI/CD는 코드 변경을 자동으로 빌드, 테스트, 배포합니다."

**3분**: "자주 통합하여 충돌을 줄이고, 자동 테스트로 품질을 보장합니다."

#### 꼬리 질문
- CI vs CD vs CD?
- 배포 전략?
